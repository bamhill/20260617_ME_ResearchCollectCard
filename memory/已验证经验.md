# 已验证经验

## 数据源

### CrossRef API 替代 arXiv 期刊搜索
- 日期：2026-06-19
- 来源：ResearchCollectCard
- 结论：arXiv 的 `all:"期刊名"` 和 `jr:"期刊名"` 对期刊论文全部返回 0。CrossRef `api.crossref.org/journals/<ISSN>/works` 免费开放、跨出版社、JSON 返回完整元数据（title/authors/DOI/date）。
- 缺点：只有正式出版日期，缺 Online First，延迟 2-4 周；无摘要字段。
- 适用场景：任何需要按期刊名检索论文的项目。

## 热点提取

### YAKE 替代 KeyBERT（离线关键词提取）
- 日期：2026-06-19
- 来源：ResearchCollectCard
- 结论：HuggingFace 在受限网络不可达时，YAKE 是有效的 KeyBERT 替代方案。纯 Python、零模型下载、不需要 GPU。英文关键词提取效果可接受。
- 代价：语义质量略低于 KeyBERT，但对从摘要中提取技术术语这种任务足够。
- 适用场景：任何需要离线英文关键词提取的项目。

## CLI 设计

### 多标签 AND 筛选
- 日期：2026-06-19
- 来源：ResearchCollectCard
- 结论：`--topic` 空格分隔多标签做 AND 交集（`topic.split()` → `all(t in source.tags for t in topics)`），比单标签有效得多。配合 `--source` 两级筛选，覆盖 90% 的使用场景。
- 适用场景：任何需要多维标签过滤的 CLI。

## 工具使用

### gstack browse 路径大小写
- 日期：2026-06-19
- 来源：ResearchCollectCard
- 结论：Windows 下 gstack browse 对 `d:/` 和 `D:/` 路径有区别，截图路径必须用大写驱动器字母。
- 适用场景：Windows + gstack。

### Python 替代 Edit 处理中文文件
- 日期：2026-06-19
- 来源：ResearchCollectCard
- 结论：Edit 工具含中文文件反复编码冲突。Python `c = open(f).read(); c = c.replace(old, new); open(f,'w').write(c)` 稳定可靠。
