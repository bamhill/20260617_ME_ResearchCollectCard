ResearchCollectCard · 应用范式

三大典型使用场景 · 操作步骤 · 效果 · 适用范围

范式一：课题组日常晨报

适用场景

导师/博士生每天早上出卡，截图发课题组群，让大家跟上最新文献。

效果

从手动刷arXiv+期刊1.5小时，压缩到3分钟。学生不用各自刷，统一看日报即可。

Step 1
启动WeRSS
docker start wewe-rss
（仅公众号需要）
→
Step 2
生成卡片
python cli.py paper
python cli.py wechat
→
Step 3
截图分享
双击out/*.html
Ctrl+P 存PNG
发课题组群

范式二：方向聚焦 — 只看对口期刊+特定领域

适用场景

做可解释性方向的研究生，只想看ABS3以上+CS领域+可解释性标签的期刊论文。

效果

从32本期刊+arXiv中精准筛出10-15篇，不浪费精力在无关论文上。

python cli.py paper --source journal --topic 可解释性 --date 2026-06-17

范式三：区间回顾 — 本周+上周对比

适用场景

组会前生成一周汇总，看看这周研究方向有什么新趋势。

效果

区间自动去重合并，一张卡片覆盖多天，热点词跨天聚合。

python cli.py paper --from 2026-06-10 --to 2026-06-17 --source journal --topic CS

范式四：新来源扩展

+ CrossRef 期刊

在 sources.json 的 paper_sources 中添加：{"type":"crossref", "query":"ISSN号", "tags":["期刊","领域"]}
自动归入 journal 源组

+ RSS 直连

在 sources.json 的 paper_sources 中添加：{"type":"rss", "query":"RSS_URL", "tags":["期刊","领域"]}
自动归入 journal_sources()

不适场景

⚠ 不适合的情况

需要全文检索/引文分析 → 用 Zotero/Semantic Scholar
需要AI摘要/翻译 → 当前仅翻译标题，不做全文摘要
需要定时任务 → 暂无 cron 集成，需手动执行
需要GUI界面 → 纯CLI+HTML输出

核心设计原则： 数据驱动配置(sources.json) → 策略派发(Fetcher) → 流水线加工 → 零CDN输出。新增源只需编辑JSON，不改代码。
