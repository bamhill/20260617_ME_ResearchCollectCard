# 方案探索过程.md

## 2026-06-17 — 从头脑风暴到定案

### 方案对比

| 方案 | 描述 | 结论 |
|:--|:--|:--|
| A: 纯静态 HTML + Python 脚本 | MyPlatform 模式，JSON + HTML 渲染 | 可用但缺管理 |
| B: Streamlit 管理后台 | Web GUI 管理来源和录入 | 过度工程，file:// 不可用 |
| **C: Python CLI + HTML 输出** | typer CLI + Jinja2 模板 + JSON 存储 | ✅ 选用 |

### 灵感来源

- [Trae 论坛 Skill 创作赛帖](https://forum.trae.cn/t/topic/18195)：杂志风竖版 HTML 日报 + Save PNG + 离线可用
- 参考了《纽约时报》《经济学人》排版：封面→导语→主题版块→快讯→来源页脚

### 关键转折

1. **两卡合并→独立生成**：最早设计一个页面含论文+公众号两区，改为 `python cli.py paper` 和 `python cli.py wechat` 独立生成，按需选择。

2. **RSS 来源方式**：讨论手动录入 vs RSS 桥接 vs 混合。定案用 WeRSS / Feeddd / RSSHub 做 RSS 桥接，来源配置数据驱动（sources.json）。

3. **idea 库定位**：从"本项目内置 idea 库"改为"卡片产出 JSON → MyPlatform scan.py 扫描 → workbench 展示"。本项目只负责产出和选择，MyPlatform 负责积累展示。

4. **单页→三页分离**：最早 magazine 卡片内含复选框选 idea，改为三页架构——dashboard 主编桌 + 干净杂志卡片页 + 独立 idea 选取面板。卡片可干净分享给学生，选取面板在 dashboard 内滑出。

5. **日期范围支持**：CLI 支持 `--date` 单日和 `--from/--to` 范围。

### 淘汰方案

- 卡片内嵌 idea 复选框（污染分享页，改为独立面板）
- Streamlit 方案（file:// 不可用，过度工程）
- 单文件架构（职责不清，拆为分层架构）
