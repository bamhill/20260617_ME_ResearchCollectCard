# 需求变更记录.md

## 2026-06-17 — v1 开发中的需求演化

| # | 变更 | 原因 | 影响 | 决策 |
|:--|:--|:--|:--|:--|
| 1 | 两卡合并→独立生成 | 用户希望按需生成某一张 | CLI 拆为 paper/wechat 两个命令 | ✅ |
| 2 | 手动录入→RSS 桥接 | 自动化优于手动 | wechat 来源改为 feedparser 读 RSS | ✅ |
| 3 | 单页含复选框→三页分离 | 卡片要干净分享给学生，选择要独立 | 新增 index.html dashboard + idea 面板，卡片模板去复选框 | ✅ |
| 4 | CLI 入口→浏览器入口 | 用户要有个打开页面，不是敲命令 | 新增 index.html 主编桌 dashboard | ✅ |
| 5 | summary_zh 子类重复→提升到基类 | Code review 发现 | RawItem 增加 summary_zh，PaperItem/WechatItem 移除重复字段 | ✅ |
| 6 | items key 冲突→改 entries | Jinja2 中 section.items 被解析为 dict.items() | 模板和 render.py 中 key 统一改为 entries | ✅ |
| 7 | is_idea 字段新增 | 用户要能选择哪些条目进 idea 库 | RawItem 加 is_idea: bool=False，Card/CardMeta 加 idea_count | ✅ |
| 8 | GBK 编码冲突 | Windows 终端 GBK 无法编码 ✓ | 所有 CLI 输出用 [OK] 替代 Unicode 符号 | ✅ |
