# 方案架构设计过程.md

## 2026-06-17 — v1 架构定案

### 分层设计

```
cli.py (typer 薄壳 + 依赖注入)
  ↓
pipeline/ (编排层)
  FetchPipeline → TransformPipeline → RenderPipeline
  ↓                    ↓
fetchers/           store/
(策略模式)          CardRepository
  ↓
models.py (Pydantic v2)
```

### 关键决策

1. **AbstractFetcher 策略接口** — arXiv / bioRXiv / RSS 三种来源各实现一个 Fetcher。新增来源 = 实现接口 + sources.json 加一行，Pipeline 和 CLI 不动。

2. **依赖注入** — cli.py 组装对象图：`FetchPipeline(fetchers={...})`，各模块只依赖抽象接口，不直接创建实例。

3. **三阶段流水线** — Fetch（采集）→ Transform（去重+排序）→ Render（Jinja2→HTML），每层独立可测。

4. **file:// 直开** — HTML 零 CDN 依赖，html2canvas 内联 base64，CSS/JS 全部自包含。

5. **三页分离** — index.html（dashboard 主编桌）+ out/paper_*.html / out/wechat_*.html（干净杂志卡片）+ idea 选取面板（dashboard 内置滑出面板）。

6. **数据模型** — RawItem 含 is_idea 开关，summary_zh 提升到基类消除子类重复。Card.idea_count 属性自动统计。

7. **GBK 兼容** — CLI 输出用 `[OK]` 代替 Unicode 符号，避免 Windows GBK 终端 crash。

### 技术栈

Python 3.10+ / typer / pydantic v2 / jinja2 / feedparser / httpx / pytest
