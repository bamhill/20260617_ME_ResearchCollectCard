## 1、Skill 简介

**科研新资讯卡片生成器 (ResearchCollectCard)** 是一个能从 arXiv 顶会/CS 分类、CrossRef 32 本 ABS/FMS/CAS 期刊、RSS 公众号三路自动抓取最新论文，生成复古编辑部杂志风竖版 HTML 日报的 Claude Code Skill。

你只需要说一句"生成今天的科研日报"或"出 ABS3 管理期刊可解释性卡片"，它就能自动抓取论文、提取热点主题、整理中文摘要、渲染成一份可以直接分享的杂志风日报，底部还带"Save PNG"按钮，一键保存为图片。

**适合谁用**：每天想快速了解 AI/工程/管理领域动态的研究生、青年教师、课题组，以及任何想定期做文献追踪、但又不想每天刷 arXiv + 逐本翻期刊的人。

## 2、使用场景

### 为什么想做它？

一个高校课题组/研究生的真实日常：

- **早上 9 点**：打开 arXiv，翻 cs.CL、cs.AI、cs.LG…
- **中午**：挨个打开 KBS、TKDE、EAAI 期刊主页，看最近上线了什么
- **下午**：翻几个公众号（NLP、专知），看看有没有错过什么
- **晚上**：想把今天看到的内容整理成简报，但手动排版要 1 小时

### 之前遇到的麻烦

| 痛点 | 具体表现 |
|------|---------|
| 信息源分散 | arXiv、CrossRef、RSS 公众号……要刷 5+ 个平台 |
| 期刊太多 | ABS3/4、FMS A/B、CAS 1/2……30+ 本，不可能每天翻 |
| 排版耗时 | 手动整理 + 排版一份简报，至少 1 小时 |
| 热点提炼难 | 52 篇论文，不知道今天的热点是什么 |
| 分享不便 | 纯文字没人看，长图又不好做 |

### 做出来之后省掉了什么

| 之前 | 现在 |
|------|------|
| 刷 5+ 平台每天 30 分钟 | 一条命令，三路聚合 |
| 翻 30+ 期刊 | `--source journal --topic CAS1` |
| 手动排版 1 小时 | 杂志风模板，秒级渲染 |
| 不知道热点 | 自动提取 + 分类（主题 / 方法） |
| 不好分享 | Save PNG 按钮，一键保存图片 |

**一句话总结**：从"刷 30 分钟资讯 + 逐本翻期刊 + 排版 1 小时"到"一句话生成杂志风日报"，每天省出 1.5 小时。

## 3、创作过程

### 核心思路

整个 Skill 的架构分 4 层：

```
数据层 → 处理层 → 热点提取 → 渲染层
arXiv/crossref/RSS → 去重+排序+合并 → YAKE+DeepSeek → Jinja2杂志风HTML
```

**关键创新**：不是"摘要翻译"，而是从当日全量论文中**自动提取主题热点 + 技术方法热点**并分组展示。

### Step 1：选择数据源

对比了几个论文聚合平台：

| 平台 | 优点 | 缺点 |
|------|------|------|
| arXiv API | CS 学科覆盖全，顶会预印本多 | 搜不到期刊名 |
| CrossRef API | 跨出版社、ISSN 精准、免费 | 只有正式出版日期，缺 Online First |
| RSS 官网 | 含期刊 RSS、公众号 RSS | 大批量被 403/418 拦截 |
| Semantic Scholar | 摘要完整 | 需 API key，频率限制严 |

最终选择 **arXiv + CrossRef + RSS 三路并行**：
- arXiv：顶会（ICML/NeurIPS/ICLR/ACL/CVPR/AAAI）+ CS 学科分类
- CrossRef：32 本 ABS/FMS/CAS 期刊，ISSN 精准匹配
- RSS：3 个公众号源（通过 WeWe RSS Docker 转发）

### Step 2：设计内容结构

参考了 AI Daily 参考设计 + 杂志排版原则：

```
封面(kicker+标题+团队+日期) → 热点区(主题/方法双列) → 论文清单(按标签分组)
```

热点区是核心特色：

```
┌── 热点主题 ────────┬── 今日热点方法是什么 ──┐
│ · LLM · 图分析     │ ▸ 近端策略优化 (2)      │
│ · 多模态 · 可解释性  │ ▸ 梯度知识蒸馏 (1)      │
│ ── 本期聚焦... ──── │ ▸ 扩散模型 (5) ▸       │
│ 大型语言模型在哲学、 │   ┌ 匹配论文列表 ┐      │
│ 法律与健康领域的应用  │   │ Zone of Proximal│  │
│                     │   └────────────────┘   │
└────────────────────┴────────────────────────┘
```

每个方法标签点击展开对应论文清单，可跳转原文。

### Step 3：热点提取——从摘要中自动抓今日焦点

**技术路线**：YAKE（无需模型下载的英文关键词提取）→ DeepSeek API 翻译+分类

```
52 篇论文摘要 → YAKE 提取 top 15 英文关键短语
              → DeepSeek："分为研究主题和技术方法，翻译中文"
              → 输出：themes=["大语言模型", "多模态对齐"]
                       methods=["扩散模型", "近端策略优化"]
              → 方法→论文匹配（英文短语回溯匹配）
```

### Step 4：多标签交叉筛选

不是"按单一标签过滤"，而是 **AND 交集**：

```bash
python cli.py paper --source journal --topic ABS4 管理信息系统
# 筛选条件：type=crossref AND 有"ABS4"标签 AND 有"管理信息系统"标签
```

这让用户能精确地"只看 ABS3 级工程类期刊"或"只为 CAS1 可解释性论文出卡"。

### Step 5：杂志风视觉设计

这是最花心思的部分。目标是：**让 HTML 看起来像纸质杂志，而不是网页**。

设计语言：
- 米色纸张背景（#e9e2d6）— 模拟纸张质感
- 深墨色文字（#15130f）— 印刷感
- 红黑强调 — 经典杂志配色（#d7382f）
- 衬线体标题 — 杂志封面感
- 虚线分隔 + 留白 — 版块节奏
- 分区标题红底白字 — 信息层级明确

CSS 设计规则：
- 对比度 ≥5.3:1（WCAG AA）
- 红色仅用于结构性强调，不在琐碎标签上撒
- 全局连续编号，不重置

## 4、使用步骤

### 安装

```bash
git clone <repo-url>
cd ME_ResearchCollectCard
pip install -r requirements.txt
```

### 每日使用

```bash
# 今日顶会日报（默认 6 源 ICML/NeurIPS/ICLR/ACL/CVPR/AAAI）
python cli.py paper

# CS 类论文
python cli.py paper --source cs

# 期刊精选（32 本 ABS/FMS/CAS）
python cli.py paper --source journal

# 中科院 1 区期刊
python cli.py paper --source journal --topic CAS1

# ABS3 管理信息系统期刊
python cli.py paper --source journal --topic ABS3 管理信息系统

# 公众号日报
python cli.py wechat

# 日期区间（自动合并为一张卡）
python cli.py paper --from 2026-06-10 --to 2026-06-18 --source journal --topic 可解释性
```

## 7、总结与思考

### 效率提升

| 指标 | 之前 | 现在 | 提升 |
|------|------|------|------|
| 信息来源聚合 | 刷 arXiv+期刊+公众号 30 分钟 | 三路自动聚合 | **30x** |
| 期刊筛选 | 手动翻 30+ 本期刊目录 | `--topic ABS4 管理` | **∞** |
| 热点提炼 | 人工浏览摘要后总结 | YAKE+DeepSeek 自动提取 | **免人工** |
| 排版设计 | 手动排版 1 小时 | 杂志风模板秒级渲染 | **360x** |

### 最满意的地方

1. **热点自动提取**：不是简单翻译摘要，而是从全量论文中统计高频关键词再分类为主题/方法，有信息增量
2. **多标签交叉筛选**：ABS4+管理、CAS1+可解释性，像数据库查询一样精准定位
3. **杂志风视觉**：米色纸张 + 衬线字体 + 红黑强调，真的有"纸质杂志"的感觉
4. **完全离线**：HTML 双击即开，不依赖任何 CDN
5. **期刊覆盖广**：CrossRef ISSN 精准匹配，同一条 API 覆盖所有出版社

### 踩过的坑

1. **arXiv 搜不到期刊名**：一开始试图在 arXiv 里用 `all:"Knowledge-Based Systems"` 搜期刊，全 0。才意识到 arXiv 只收预印本、不索引期刊名，最终走 CrossRef API。
2. **CrossRef 只有正式出版日期**：期刊上线到 CrossRef 收录有几周延迟——我们 6 月只能搜到 5 月底的论文。RSS 直连出版社做补充是更准确的路线。
3. **热点提取不能依赖模型下载**：本想在热点提取中用 KeyBERT，但 HuggingFace 在此网络环境不可达。换成 YAKE（纯 Python、零模型下载），效果达标。

### 后续优化方向

- RSS 直连出版社期刊（解决 CrossRef 延迟问题）
- 支持更多学科主题标签扩展
- Bohrium API 对接（用户提到的学术聚合平台）
- MyPlatform 集中管理面板
