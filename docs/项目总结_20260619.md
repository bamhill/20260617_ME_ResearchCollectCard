# ResearchCollectCard 项目总结

> 2026-06-17 立项 → 2026-06-19 收尾，3天迭代

## 项目目标

**初衷**：课题组每天手动刷 arXiv + 期刊主页 + 公众号，至少 1.5 小时。做一个 CLI 工具自动聚合，输出杂志风 HTML 日报。

**实际产出**：66 源多路聚合（arXiv 8 + CrossRef 32 + 期刊RSS 19 + 公众号 7），三阶段流水线（fetch→transform→render），杂志风 HTML + YAKE/DeepSeek 热点提取，Claude Code Skill 发布，GitHub 公开，小红书推广。

## 技术方案

- **语言**：Python 3.10+
- **CLI**：typer
- **数据模型**：pydantic v2
- **模板**：Jinja2
- **Fetcher 策略**：AbstractFetcher → ArxivFetcher / CrossrefFetcher / RssFetcher
- **热点提取**：YAKE（关键词）+ DeepSeek API（翻译分类）
- **RSS**：feedparser，Elsevier summary 日期回落，TOC 过滤
- **输出**：自包含 HTML，file:// 直开，零 CDN

## 关键决策

1. **RSS 双源策略**：期刊 RSS 直连抢时效 + CrossRef API 保底，URL 去重
2. **严格日匹配**：RSS 不设时间窗口，按论文固定出版日精确匹配
3. **数据驱动**：sources.json 配置一切，新增源不改代码
4. **out_ 文档体系**：8 件 HTML（基础5+工程3），深色/浅色双主题

## 踩坑记录

1. **Elsevier RSS 日期在 summary 不在标准字段**：加 `_parse_date_from_summary()` 回落
2. **IEEE RSS TOC 占位条目**：加 `_is_meta_entry()` 过滤
3. **T&F/Wiley/INFORMS/ACM 等 RSS 被 Cloudflare WAF 拦截**：暂无法直连，保持 CrossRef 单源
4. **公众号 feed ID 多次试错**：WeRSS 后台 ID 和实际内容对不上，需逐一验证
5. **GitHub 网络不通**：部分 commit 无法 push，待通网补推

## 经验提炼（可跨项目复用）

1. **策略模式 + 配置驱动**：AbstractFetcher → sources.json → FetchPipeline 自动派发，新增源零代码改动
2. **out_ 文档自动生成**：模板 + CSS 主题 + 13 类骨架 → Python 脚本批量生成，已沉淀为 out-generator Skill
3. **gstack 截图流程**：HTML → localhost server → gstack goto → screenshot → Python 合成
4. **Skill 应有 Python 脚本**：纯提示词 Skill 每次重新推理浪费 token，脚本直接执行可复用
